###########################################################################################################################################
#######################################################  PLANNING   #######################################################################
###########################################################################################################################################

Upon receiving a 2D spatial map (file.nc) containing the uncertainty/error values of a specific physical quantity (currently: observation error - temperature [Celsius]),the
objective of the Optimal Planning algorithm is to generate trajectories for N vehicles with specified endurance [hours] and speed (m/s) that 
maximize the total uncertainty/error collected along the trajectories.
Input of the Algorithm: 2D map - Output of the Algorithm: routes_file.txt containing the waypoints of the optimal trajectories.

After a preliminary insertion of masks (land/sea - minimum depth - minimum distance from land - obstacles, ...) on the map to delineate the free area of navigation, 
the algorithm consists of two main steps:

|* Point Generation: Generation of points of interest within the grid containing the uncertainty values.
|* Prize-collecting Vehicle Routing Problem: to optimally connect and select the identified points (clients).

|* Point Generation:
To generate the points, contour lines are derived from the map, and points maintaining a specified minimum distance between them are selected 
along these lines. Then, the Voronoi diagrams of the selected points is calculated to identify equidistant points from the already identified 
ones, thereby covering the areas inside the contour lines. The Voronoi step is applied multiple time (adaptive version of this step is actually under development)

|* Prize-Collecting Vehicle Routing Problem:
To solve the Vehicle Routing Problem, the PyVRP solver was chosen: an open-source Python solver.
(PyVRP: A High-Performance VRP Solver Package https://pubsonline.informs.org/doi/10.1287/ijoc.2023.0055).
The solver supports multiple vehicles, multiple depots, maximum distance constraints, and the prize-collecting variant. 
In this variant, the nodes (clients) are not equal; each node has a prize, and the goal is to balance the competing objectives of minimizing travel 
costs while maximizing the collected prizes. While respecting the constraints.
In the algorithm, each point (client) is assigned a prize proportional to the error, in order to maximize the collection of points with large errors.

Note:
The setup of the solver does not take into account the time spent visiting each client (which is translated as waiting time 
at each waypoint before moving to the next one). As a result, the solution would exceed the maximum mission time desired and set in the Config.file.
To address this issue, in our algorithm the PyVRP solver is applied twice. The first time, it aims to estimate (approximately) the number of clients each vehicle can 
visit within the available mission time. For each visited client, a waiting time specified in the Config.file is considered, and this time is subtracted 
from the desired mission time.
At this point, the solver is applied again, and the final routes that respect the operation time (travel time + waiting time) are generated.

Once the waypoints for each route are identified, a check is performed to know the minimum depth for each route. 
This information, along with travel time, distance, waypoint coordinates, and other data, is saved in the file routes_file_with_wt.txt, which must then 
be parsed for the actual mission creation in Neptus.

#################################################
############### Code organization ############### 
#################################################

OptimalPlanning.py > Algorithm

Dataset.nc > Model's output file in NetCDF format: file containing the following fields: temperr2d, latitude, longitude, tbath, land_mask for the surface level.

Config_file.py > Configuration file where to set: Mission parameters - Mask parameters - Parameters for point of interest selection - Solver parameters

Utils.py > Utils function for the algorithm

VRP_instance_problem.vrp > file automatically generated by the algorithm. It contains the instance for the Vehicle Routing Problem.
                           (number of clients, numero of vehicles, starting and ending points, distance matrix)

routes_file_node_estimation.txt > Output of the algorithm. File containing the waypoints of the optimal routes and other info (solution of the first istance. Client estimation solution)

routes_file.txt > Output of the algorithm. File containing the waypoints of the optimal routes and other info (solution of the second istance. Final solution)

plots > contains the final plot of every run.
